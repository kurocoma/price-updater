# 商品価格改定ツール — レビュー用プロンプト

以下の要件定義と実装計画について、抜け漏れ・リスク・改善点をレビューしてください。

---

## 背景

ネクストエンジン（NE）で全モールの受注管理を行っているEC事業者が、商品の価格改定を効率化したい。
現状は各モールの管理画面に個別ログインして手作業で価格変更しており、工数がかかっている。

## 目的

NEの商品マスタをベースに、楽天・Yahoo・Shopify・NE自体の商品価格を一括改定するWebツールを開発する。

## 対象モール（Phase 1）

| モール | 店舗名 | API |
|---|---|---|
| 楽天市場 | おきなわ一番 楽天市場店 | RMS API (ItemAPI) |
| Yahoo!ショッピング | くすりの健康家族 | ストアAPI (editItem / updateItems) |
| Shopify | 株式会社くりま公式ショップ | Admin GraphQL API (productVariantsBulkUpdate) |
| ネクストエンジン | 自社 | NE API (/api_v1_master_goods/upload) |

※ Amazon は Phase 2 で対応（保留）

## 商品コード体系

### 単品商品（syohin_basic CSV）
- `syohin_code`: 商品コード（一意キー）
- `baika_tnk`: 税抜売価
- `tax_rate`: 税率（8% or 10%）
- `zaiko_su`: 在庫数

### セット商品（set_syohin CSV）
- `set_syohin_code`: セット商品コード
- `set_baika_tnk`: 税抜セット売価
- `syohin_code`: 構成単品の商品コード
- `suryo`: 構成数量
- 1つのset_syohin_codeに複数のsyohin_code × suryoが紐づく

### 各モールとの商品コード紐づけ
- 原則: NEの `syohin_code` = 各モールの商品コード
- 例外: `himoduke` CSVにモール別の別コードが記載されている場合、そちらを使用
- セット商品の `set_syohin_code` は各モールでも同じコード（例外なし）
- ただし全モールに全商品が存在するとは限らない

### 各モールの商品コード列と価格列
| モール | 商品コード特定方法 | 価格列 | 価格形式 |
|---|---|---|---|
| 楽天 | `システム連携用SKU番号`（空なら`商品番号`） | `通常購入販売価格` | 税抜 |
| Yahoo | `code` | `price` | 税込 |
| Shopify | `Variant SKU` | `Variant Price` | 税込 |
| NE | `syohin_code` | `baika_tnk` | 税抜 |

## 価格計算ロジック

- マスター価格: ユーザーが入力する価格は**税抜**
- 楽天 / NE: そのまま税抜で反映
- Yahoo / Shopify: `税抜価格 × (1 + tax_rate/100)` → **切り捨て**で税込に変換

## ワークフロー

1. **データ準備**: NE管理画面からCSV手動ダウンロード → ツールにインポート → SQLiteに格納
   - syohin_basic CSV（単品商品）
   - set_syohin CSV（セット商品）
   - himoduke CSV（紐づけマッピング）

2. **価格改定**:
   - ユーザーがCSV（`syohin_code, new_price`）をアップロード
   - ツールが各syohin_codeに関連するset_syohin_codeを自動洗い出し
   - セット商品にはユーザーが個別に売価を設定
   - プレビュー画面で「現在価格 → 新価格」を確認（ON/OFF切替可能）
   - 反映対象モールを選択可能

3. **バックアップ**: API反映前に変更前価格を自動でローカルCSV保存（`backups/YYYY-MM-DD_HHmmss/`）

4. **API反映**: 選択モールに順次反映、モール別に独立実行

5. **エラー処理**: 失敗モールのみ個別リトライ可能

## 技術スタック

| 項目 | 選定 | 理由 |
|---|---|---|
| フレームワーク | Next.js (App Router) + TypeScript | 他スタッフも利用する可能性あり → Web UI。既存ツールと統一 |
| UI | shadcn/ui + Tailwind CSS | テーブル表示・フォーム入力に最適 |
| DB | SQLite（ローカル） | 必要な時だけ起動。Supabase不要 |
| ORM | Drizzle ORM | SQLite相性良好、軽量 |
| API連携 | Next.js Route Handlers | APIキーをブラウザに露出させない |
| 認証情報管理 | .env ファイル | ローカル保存 |

## 利用環境

- Windows 11 ローカルPC
- 必要な時だけ起動
- 他スタッフも使用する可能性あり
- 1回の操作で数十件を一括改定

## NE API 補足

- OAuth2認証（client_id / client_secret / access_token / refresh_token）
- 商品マスタ更新はCSVアップロード → 非同期処理
- 月1000回以下 & 3GB以下は無料
- テスト環境のキーは取得済み。本番は審査承認後に利用可能

